## 一、设计风格规范（Design Tokens，全局统一）
- **页面背景（基底）**：极简浅灰白 `#f4f1ec`（低饱和米白，非纯白）
- **卡片 / 左右栏背景**：纯白 `#ffffff`
- **主色（暖橙）**：`#d98841`；浅橙底（选中态 / 按钮）`#fbe9dd`；hover `#fad4be`
- **文字**：主文深炭灰 `#2a2520`；辅助文**深灰** `#3d3a36` / `#4a4540`（**不用浅灰，保持清晰可读，无刺眼高对比**）
- **描边**：细浅米色 `#e5e0d8`（卡片外框） / `#e6e0d6`（卡片内分割线）
- **圆角**：大卡片 `20px`，药丸 / 按钮 `14px`，小元素 `10–12px`
- **阴影**：`0 2px 16px rgba(54,44,34,0.06)`（极淡悬浮感，不沉重）
- **字体**：系统无衬线（PingFang SC / 微软雅黑）；页面标题 `24–30px / 800`，板块标题 `17px / 800`，正文 `14–16px`，辅助文字 `13–14px 深灰`
- **图表色**：主系列暖橙 `#d98841`；多系列用低饱和莫兰迪柔和色系（雾蓝 `#9bb4c4`、灰绿 `#a9bda3`、藕粉 `#d9b8b0`、浅紫 `#bdb0cf` 等）
- **整体气质**：极简浅色、留白充足、现代简约商务 SaaS 风；**低饱和、无高对比度刺眼色块**

## 二、整体布局（三栏轻量化后台 Dashboard）
- **左栏（固定白色圆角卡片，上下铺满整页）**：品牌区 + 导航单选菜单，常驻不动
- **中栏（浅灰白主内容区）**：当前视图内容，随左导航切换
- **右栏（固定白色圆角卡片，上下铺满整页）**：业务快捷栏，全局常驻
- 三栏之间留白充足；左右白卡与中栏浅灰白底形成柔和分层（靠色差 + 极淡阴影，不靠粗重边框）
- **无顶栏设置入口、无 API 配置页**

## 三、产品一句话定位
面向药企 **BD（商务拓展）团队** 的 AI 竞品监测驾驶舱：输入治疗领域 / 靶点 / 公司，自动聚合全球（ClinicalTrials.gov + 中国 CDE / ChiCTR）临床试验数据，输出竞争格局、近期动态、竞品对比与 AI 简报，辅助 BD 决策。

## 四、目标用户与场景
- **用户**：药企 BD 经理、竞品情报分析师、投资 / 战投研究员
- **核心场景**：每周例会前 5 分钟生成领域看板；竞品新药临床进展追踪；中国管线与 CDE 审批监控；向领导汇报时一键出 AI 简报

## 六、页面详细定义（7 个视图，均不含 API / 模型配置）

### 页面 1 · 总览
- **顶部**：页面标题「总览」(28px/800) + 副标题「一键聚合治疗领域的试验规模、竞争格局与近期动态」（深灰 14px）
- **输入区**：文本框「监测主题（治疗领域 / 靶点）」占位符「如 NSCLC, PD-1, CAR-T」；主按钮「生成看板」（暖橙填充，type=primary）
- **指标卡（4 个并排，白底圆角卡片）**：总试验 / 活跃试验 / 近 30 天新增 / 本周新增（大数字 34px / 800 深炭灰 + 标签 13px 深灰）
- **图表区**（图表标题 17px/800，居中）：
  - 水平条形图「主要申办方 Top」（暖橙单色，y=申办方，x=试验数）
  - 柱状图「阶段分布」（莫兰迪多色，x=阶段 Phase1/2/3/4/N.A，y=试验数）
  - 面积折线图「近 30 天时间趋势」（暖橙线 + 浅橙填充）
- **列表「近期重要更新」**：试验卡列表（每条：标题可点击链到原始协议 + 元信息「NCT号 · 阶段 · 状态 · 日期」+ 可选风险标签彩色 chip）
- **底部按钮**：「★ 加入我的监测」（全宽浅橙按钮）
- **空态**：未生成时居中深灰提示「输入监测主题并点击生成看板生成数据看板」

### 页面 2 · 竞争格局
- **顶部**：标题「竞争格局」+ 副标题「分析治疗领域的竞争密度：主要玩家与阶段分布」
- **输入区**：文本框「治疗领域」+ 文本框「聚焦申办方（可选）」；主按钮「生成分析」
- **指标卡（4 个）**：总试验 / 活跃试验 / 主要申办方 / 中国药企
- **图表区**：主要申办方 Top 水平条形图 + 阶段分布柱状图（同总览样式）
- **按钮**：「★ 加入我的监测」「生成 AI 竞争简报」
- **AI 简报区**：点击后展示中文 Markdown 文本块（主要玩家管线布局、各阶段竞争密度、空白领域、BD 策略建议）

### 页面 3 · 试验检索
- **顶部**：标题「试验检索」+ 副标题「按疾病 / 靶点 / 公司检索 ClinicalTrials.gov」
- **输入区**：文本框「疾病或关键词」（占位「NSCLC, CAR-T…」）+ 文本框「申办方（可选）」+ 下拉「状态（可选）」选项：空 / RECRUITING / ACTIVE_NOT_RECRUITING / COMPLETED / TERMINATED / WITHDRAWN；主按钮「检索」
- **结果区**：顶部「共 N 条结果」计数（深灰）；下方试验卡列表（同总览卡片样式）
- **详情区**：下拉「查看试验详情」（选 NCT ID）→ 展示 标题 / 申办方 / 阶段·状态 / 原始协议链接 / 摘要 / 主要终点列表

### 页面 4 · 每日监测
- **顶部**：标题「每日监测」+ 副标题「追踪治疗领域近期新增 / 更新试验」
- **输入区**：文本框「治疗领域」+ 数字输入「监测天数」（1–90，默认 7）；主按钮「监测」
- **指标卡**：「新增 / 更新」大数字
- **图表**：近 N 天时间趋势面积折线图（暖橙）
- **列表**：试验卡列表（最多 30 条，同总览卡片样式）

### 页面 5 · 竞品对比
- **顶部**：标题「竞品对比」+ 副标题「并排对比多个试验：设计 / 终点 / 竞争定位」
- **输入区**：多行文本框「输入 NCT ID（逗号分隔，最多 5 个）」占位「NCT04267848, NCT04191356」；主按钮「对比」
- **对比表格**（横向滚动，白底圆角容器）：列 = NCT（可点击链接）/ 申办方 / 阶段 / 状态 / 条件 / 干预 / 主要终点
- **AI 对比分析**：表格下方 Markdown 文本块（设计差异、终点优劣、竞争定位结论）
- **校验**：少于 2 个 ID 时报错提示「至少输入 2 个 NCT ID」

### 页面 6 · 中国管线
- **标签页（Tabs，药丸样式）**：「管线检索」/「CDE 审批」
- **Tab1 管线检索**：文本框「治疗领域」+ 主按钮「检索中国管线」→ 指标「中国药企试验」数 + 中国申办方列表（每项「药企名 · N 项」+ 名下试验卡）+ ChiCTR 检索链接
- **Tab2 CDE 审批**：文本框「药品名（中 / 英）」+ 主按钮「查询 CDE」→ CDE 数据查询链接 + 药物临床试验登记平台链接 + 快捷链接列表 + 关联中国试验卡列表

### 页面 7 · 智能助手
- **顶部**：标题「智能助手」+ 副标题「自由提问，Agent 自动调用工具检索并整合」
- **输入区**：多行文本框「输入查询」（高度约 100px，可预填右侧快捷查询带入的词）
- **按钮**：「▶ 运行 Agent」（暖橙主按钮）
- **结果区**：Markdown 渲染的 AI 回答（含可点击引用链接）
- **错误态**：凭证错误 / 限流 / 超时 / 工具调用失败 时显示红色错误提示条
- **展开区「Agent trace」**：折叠面板，列出每一步「Step i：工具名」+ 入参 JSON
- **注意**：本页**不含** API Key 输入框、**不含** 模型选择下拉框（密钥与模型由环境配置）

## 七、交互流程
1. 用户点左导航 7 项之一 → 中栏切换对应视图，左右栏保持常驻
2. 用户点右栏「快捷查询」按钮 → 跳转智能助手页 + 预填查询词
3. 用户点右栏「我的监测」项 → 跳转总览页 + 自动带入该监测条件并生成看板
4. 各视图输入条件后点主按钮 → loading（spinner）→ 渲染指标 / 图表 / 列表
5. 智能助手运行 Agent → 展示流式 Markdown 结果 + 可展开 trace

## 八、示例数据（建议填充 NSCLC 非小细胞肺癌领域，便于演示）
- 指标：总试验 ≈ 1200 / 活跃 ≈ 600 / 近 30 天新增 ≈ 45 / 本周新增 ≈ 12
- Top 申办方：恒瑞医药 / 百济神州 / 信达生物 / 默沙东 / 罗氏 / 阿斯利康（试验数递减）
- 阶段分布：Phase 1 ≈ 180 / Phase 2 ≈ 260 / Phase 3 ≈ 120 / Phase 4 ≈ 20 / N.A ≈ 20
- 时间趋势：近 30 天每日新增 0–5 波动
- 试验卡示例：标题「A Phase 3 Study of Osimertinib in NSCLC」/ NCT04267848 / Phase 3 / RECRUITING / 2026-06-20 / 风险标签「全球多中心」
- 中国管线示例：恒瑞医药 · 8 项；百济神州 · 6 项
