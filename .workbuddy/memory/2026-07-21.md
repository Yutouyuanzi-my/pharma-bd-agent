# 2026-07-21 工作日志

## Agent 输出改为流式（核心改动）
- 用户需求：把 Agent 输出从「一次性等待」改为「流式输出」。
- `agent.py`：新增 `run_agent_stream()` 生成器，用 `stream=True` 调 LLM，逐 chunk 累积文本 / 工具调用增量，实时 `yield` 事件：`content` / `tool_start` / `tool_done` / `final` / `error`。原 `run_agent()` 保留未删（无引用，纯增量）。
- `app.py`：新增 `_stream_agent(text_slot, status_slot, query, ...)` 辅助函数，用 `st.empty()` 占位实现「打字机」效果（每约 12 字节流刷新）+ 工具调用实时进度列表；`render_assistant()` 与 `render_landscape()`（AI 简报）两处改为调用它。import 改为 `from agent import register_tool, run_agent_stream`。

## 验证方式（关键经验）
- 沙箱内 `import openai` 会触发网络挂起（代理/无网），无法直连验证。改用「桩 openai 模块」：`sys.modules['openai']=types.ModuleType('openai')` 后再 import agent，配 mock client 跑事件循环，零网络依赖，可靠。
- 测试中发现并修复一个真实 bug：原代码 `tool_start` 的 yield 写在 `if func_name in REGISTRY` 的 else 分支里，未知工具时（LLM 拼错工具名）会**静默跳过事件**，UI 丢进度。已重构：参数解析 + `tool_start` 提到 REGISTRY 判断之前，`tool_done` 在 for 循环末尾（已知/未知通用）发出。
- 用 mock 客户端（正确注册名 `search_clinical_trials`）验证事件顺序 content→tool_start→tool_done→content→final 正确；未知工具分支也确认 emit tool_start/tool_done。
- py_compile 语法检查通过；已提交 a6f7978 并 push 到 GitHub（Streamlit Cloud 会自动重新部署）。
