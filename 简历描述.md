# 简历项目描述（推荐版本）

## 版本一：投Agent工程师岗位

---

**临床试验竞品监测Agent（AI Engineer方向）**　　　2026.07 — 进行中

面向药企BD/战略部门的竞品管线动态监测工具，基于OpenAI Function Calling从零实现Agent循环

• **Agent架构设计**：纯Python实现tool registration + LLM reasoning loop，6个专业工具（搜索/详情/文献/格局分析/变更监测/并排对比），无需第三方Agent框架

• **数据层**：集成ClinicalTrials.gov API v2（高级查询语法支持申办方/日期范围过滤）+ PubMed E-utilities，返回结构化JSON供LLM分析

• **BD场景适配**：竞争格局自动分组（按申办方/阶段）、每日变更监测（新增+更新双维度去重）、试验侧边对比（LLM生成中文分析报告）

• **前端与部署**：Streamlit宽屏UI（快捷按钮+Agent调用轨迹可视化），支持中文输出；已部署至Streamlit Cloud可在线体验

• **技术决策**：gpt-4o-mini控制成本、User-Agent伪装绕过WAF、指数退避重试处理API限流

## 版本二：投医疗AI产品经理岗位

---

**临床试验竞品监测Agent（医疗AI PM方向）**　　　2026.07 — 进行中

面向药企BD团队的AI竞品监测工具原型，直接对标丁香园「竞品监测Agent」岗位需求

• **痛点验证**：药企BD每天花2-3小时手动刷ClinicalTrials.gov追踪竞争对手管线动态，现有方案（Cortellis $10k+/年）昂贵且无实时监测能力

• **产品方案**：AI Agent自动执行"搜索→过滤→分析→报告"全流程，输出中文BD晨报（主要玩家、阶段分布、策略建议），目标节省80%人工时间

• **技术实现**：6个工具覆盖BD核心场景（竞争格局/每日监测/试验对比/文献检索）；支持按药企名/适应症/状态多维筛选；Markdown格式结构化输出

• **双技术栈验证**：Python工程版（展示底层Agent机制理解）+ Dify工作流版（展示低代码快速验证能力），同一需求两种架构

• **路线规划**：V1接入ChiCTR/EudraCT多数据源、定时简报邮件推送、个人Watchlist订阅；V2变化检测告警+飞书/Slack集成
